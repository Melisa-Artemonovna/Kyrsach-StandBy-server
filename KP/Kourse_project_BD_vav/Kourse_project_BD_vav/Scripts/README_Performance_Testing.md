# Инструкция по тестированию производительности таблицы Deals

## Быстрый старт

## Вариант A: MSSQL

### Шаг 1: Генерация тестовых данных

1. Откройте файл `05_GenerateTestData_Deals_MSSQL.sql` в SQL Server Management Studio (SSMS)
2. Замените `[YourDatabaseName]` на имя вашей базы данных
3. Выполните скрипт (F5)
4. Дождитесь завершения (обычно 1-5 минут для 100000 записей)

**Результат:** В таблице `Deals` будет 100000 тестовых записей

### Шаг 2: Тестирование производительности

1. Откройте файл `06_PerformanceTest_Deals_MSSQL.sql` в SSMS
2. Замените `[YourDatabaseName]` на имя вашей базы данных
3. Выполните скрипт (F5)
4. Скрипт автоматически:
   - Удалит все индексы (кроме PRIMARY KEY)
   - Выполнит тесты БЕЗ индексов
   - Создаст индексы
   - Выполнит тесты С индексами
   - Покажет сравнительную таблицу результатов

**Результат:** Вы увидите время выполнения каждого запроса и коэффициент ускорения

### Шаг 3: Просмотр плана выполнения

Следуйте инструкциям в файле `07_ExecutionPlan_Guide_MSSQL.md`

**Краткая версия:**
1. В SSMS нажмите **Ctrl + M** (включить план выполнения)
2. Выполните запрос
3. Откройте вкладку **"Execution Plan"**
4. Сравните планы до и после создания индексов

## Ожидаемые результаты

### Без индексов:
- Время выполнения: **100-500 мс** на простой запрос
- План: **Table Scan** или **Clustered Index Scan**
- Стоимость: **высокая** (> 100)

### С индексами:
- Время выполнения: **1-10 мс** на простой запрос
- План: **Index Seek**
- Стоимость: **низкая** (< 1)
- **Ускорение: 10-100x**

---

## Важные замечания

1. **Перед тестированием** убедитесь, что у вас есть резервная копия базы данных
2. **Генерация данных** может занять несколько минут
3. **Тестирование** выполняется на реальных данных, поэтому результаты могут варьироваться
4. **Очистка кэша** (`DBCC DROPCLEANBUFFERS`) выполняется автоматически для точных результатов

## Устранение проблем

### Ошибка: "Недостаточно памяти"
- Уменьшите количество генерируемых записей (измените `@TotalRecords`)

### Ошибка: "Timeout expired"
- Увеличьте timeout в SSMS: Tools → Options → Query Execution → SQL Server → General → Execution time-out

### Медленная генерация данных
- Используйте транзакции для пакетной вставки (уже реализовано в скрипте)
- Увеличьте размер пакета (`@BatchSize`)
